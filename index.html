Hansa Teutonica

Hansa Teutonica is a German Euro-style boardgame. Although it features a heavy amount of abstraction, it's surprisingly fast paced and has a substantial amount of player interaction compared to its peers.

Although a digital version already exists, I chose to implement HT in browser as I needed a project I felt passionate about after the emotional blow of getting laid off. I chose to work with minimal frameworks and tools and use vanilla JS. The game logic all occurs within the client, the backend was built with node and express and is responsible for orchestrating room creation and player turn messaging. I do all the real time messaging via websockets as I needed bi-directional communication between the server and the client.

The server is hosted in a docker container on Google Cloud Run. I found GCP to be very powerful, but frustratingly difficult at times.

I haven't touched it for a few weeks as it's fully functional right now and a complete game can be played either offline or with other players. There's a lot of things that could be worked on if I come back to it in the future. It's not very visually appealing and there's a ton of room for UI improvements as I was more concerned with getting an MVP out before touching the user experience. There's plenty of room for under-the-hood improvements as well, from input validation, to using a CDN and a bundler instead of serving files directly from my server.

I feel this project was quite valuable as a learning experience. I learned a lot about the value of frameworks when working without them; I found myself building my own tools for DOM manipulation that would have been easily available and thoroughly tested if I had been using a framework. I was reminded of the value of documentation for even single dev projects. My single biggest take away is the importance of building with the future in mind to minimize tech debt. I pushed up a lot of hacky code early, and it came back to bite me when refactoring.

I'm always interested in playing a game with real players!

Vue Clicker

This is an untitled, simple incremental/idle game build using TypeScript, Vue3, and Vite. I'd wanted to learn a new framework and heard good things about Vue, I also wanted to refresh my typescript knowledge. So far I love the simplicity of the Vue composition API. Vite also makes setup so much easier - it's really nice to be able to quickly build my dist and deploy it via cloudflare workers.

I'm actively working on this and have a ton more to do. I'm happy with the simple gameplay loop, but the beauty of incremental games is their ability to almost infinitely integrate more content into a relatively basic gameplay loop.
